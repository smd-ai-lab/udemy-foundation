🏗️ 1. Frameworks de base & Orchestration

Ces outils sont le "moteur" de l'agent : ils gèrent la logique de raisonnement, la mémoire, l'enchaînement des étapes et l'appel aux outils. Ce sont les plus complets et les plus utilisés.

  • LangChain Framework modulaire pour construire des applications avec des LLM. Il offre des abstractions pour les appels de modèles, la gestion du contexte et l'intégration d'outils.Très populaire et flexible. Idéal pour une large gamme d'applications, des simples chaînes aux agents complexes.126k⭐

  • LangGraphFramework pour créer des agents stateful, multi-acteurs sous forme de graphes. Permet de définir des nœuds, des arêtes et un état partagé.Parfait pour les workflows complexes avec des boucles, des branchements et des interventions humaines (human-in-the-loop).23k⭐

  • AutoGen (Microsoft)Framework pour construire des systèmes multi-agents où les agents collaborent via des conversations pour résoudre des tâches.Idéal pour la recherche, la simulation et les workflows complexes nécessitant une collaboration autonome entre agents.53k⭐

  • CrewAI Framework pour orchestrer des workflows multi-agents basés sur des rôles. On définit des agents avec des rôles, buts et outils spécifiques.Met l'accent sur la collaboration structurée. Très apprécié pour sa simplicité de prise en main.46k⭐

  • Semantic Kernel SDK d'orchestration de Microsoft, conçu pour intégrer l'IA dans des applications existantes (souvent en environnement .NET).Idéal pour les entreprises qui veulent ajouter des capacités d'IA à leurs logiciels sans tout réécrire.27.1k⭐

  • LlamaIndex Framework orienté "données" pour connecter les LLM à des sources de données structurées et non structurées. Le choix pour les applications de type RAG (Retrieval-Augmented Generation) où l'agent doit interagir avec une base de connaissances.43.2k⭐

  • Agno Framework léger qui met l'accent sur la composabilité avec des abstractions minimales.Idéal pour les développeurs qui veulent un maximum de contrôle et de flexibilité sans framework trop imposant.

  • La Machina Engine Moteur JavaScript/TypeScript "headless" (sans interface) conçu pour être intégré comme un "cerveau" dans des workflows de production.Idéal pour être embarqué dans des moteurs de workflow comme n8n. Multi-provider, pausable, et conçu pour le cloud (supporte Cloudflare R2).

  • AgentLoom Orchestrateur de workflows déterministe. Contrairement aux agents autonomes, vous définissez un DAG (graphe orienté acyclique) en YAML que l'LLM suit.Parfait pour les pipelines de production où la prédictibilité, l'observabilité (OpenTelemetry) et le contrôle des coûts (avec circuit breakers) sont prioritaires.

⚙️ 2. Moteurs de Workflow & Exécution

Si les frameworks d'orchestration définissent la logique, ces outils se concentrent sur l'exécution robuste, la résilience et l'industrialisation des tâches.

  • Temporal Plateforme open-source pour exécuter des workflows durables, fiables et évolutifs, avec reprise après panne.Idéal pour les processus métier critiques, les tâches de longue durée et les orchestrations complexes qui doivent être résilientes.

  • OpenHands (ex OpenDevin) Plateforme pour les agents autonomes d'ingénierie logicielle. Capable d'écrire du code, d'exécuter des tests et d'interagir avec un environnement de développement.Référence en matière d'agents de codage autonome.

🎨 3. Plateformes Visuelles & No-Code

Ces outils permettent de concevoir des workflows d'agents avec une interface graphique (drag-and-drop), sans ou avec très peu de code.

  • Dify Plateforme open-source pour construire, déployer et opérer des applications LLM et des agents. Combine orchestration, gestion des prompts, évaluation et déploiement.Solution "tout-en-un" pour aller du prototype à la production. Très complet et populaire.144k⭐

  • Flowise Builder visuel low-code pour créer des workflows LLM et agents avec une interface drag-and-drop.Idéal pour le prototypage rapide, les outils internes et l'éducation.127k⭐

  • Langflow Interface visuelle pour construire et debugger des applications basées sur LangChain.Excellent pour l'apprentissage, les démos et l'expérimentation de concepts LangChain.44.1k⭐

🧩 4. Harness pour Agents

Le terme "harness" (ou "harnais") désigne la couche qui transforme un LLM en agent prêt pour la production. Il contient la logique d'orchestration, les outils, les compétences (skills), et la gestion de la mémoire.

  • Deep Agents Deploy (LangChain)Un harnais open-source, agnostique du modèle, pour déployer des agents de façon scalable. Offre des endpoints prêts à l'emploi (MCP, A2A, memory).Alternative ouverte aux solutions propriétaires comme Claude Managed Agents. Idéal pour qui veut garder la main sur la mémoire et les données.

  • OpenClaw Harnais open-source à déployer via API. Multi-modèle, MCP-native, et offre une quarantaine de compétences intégrées.Solution flexible qui peut être hébergée sur n'importe quel cloud (ou VPS). Repose sur le framework OpenClaw.

  • Claude Code exemple populaire de "harnais", utilisant nativement les LLM Anthropic de grande qualité comme Claude-Opus.

  • Hermes Agent open-source agent framework pour executer des taches en toute autonomie, optimisation de la memoire et fonction auto-evolution.

🔌 5. Intégration & Protocoles

Pour faire communiquer vos agents entre eux et avec le reste de votre écosystème.

  • Model Context Protocol (MCP) Protocole unifié pour connecter les LLM à des outils, APIs et sources de données externes.

  • Agent-to-Agent (A2A) Protocole de messagerie sécurisée et de délégation de tâches entre agents.

💡 Comment faire le bon choix ?

Le choix dépendra fortement de votre cas d'usage.


Vous débutez ou voulez prototyper rapidement ? Commencez par une plateforme visuelle comme Dify, Flowise ou Langflow. Elles sont idéales pour se familiariser avec les concepts sans coder.


Vous avez besoin d'un framework robuste et flexible pour coder ?

  • LangChain est un choix sûr et très populaire pour une large variété de projets.

  • Pour des workflows avec des interventions humaines, des boucles complexes et une gestion d'état avancée, préférez LangGraph.

  • Si votre projet est centré sur la recherche et la collaboration d'agents multiples, AutoGen est une excellente option.

Vous devez industrialiser un processus métier stable et prévisible ?

  • Un outil comme AgentLoom, qui fonctionne sur un principe de DAG déterministe, vous offrira un contrôle total sur les coûts et l'exécution.

  • Temporal apportera une résilience inégalée à vos workflows critiques.

  • Pour une application de RAG ou de recherche documentaire ? LlamaIndex est l'outil le plus spécialisé et performant pour cette tâche.

Vous souhaitez déployer un agent "prêt à l'emploi" et exposé via API ?

  • Les harnais comme Deep Agents Deploy ou OpenClaw Managed Agents sont faits pour vous. Ils gèrent l'infrastructure et l'isolation.
